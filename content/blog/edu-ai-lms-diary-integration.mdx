## Школа -- это экосистема систем

Современная казахстанская школа использует несколько цифровых систем одновременно:

- **Электронный дневник** (Kundelik, Bilimland и др.) -- учет посещаемости, оценки, расписание
- **LMS** (Learning Management System) -- учебные материалы, задания, тесты
- **Электронный журнал** -- ведение академической документации
- **Системы отчетности** -- формы для районных и областных отделов образования

Каждая система требует своих данных, своего формата, своего ввода. Учитель **вводит одну и ту же информацию в 3-4 системы**, тратя драгоценное время на дублирование.

Добавление еще одной системы -- AI-ассистента -- без интеграции с существующими только **усугубит проблему**. Поэтому интеграция -- не дополнительная функция, а необходимость.

## Принципы интеграции Alashed EDU

### Принцип 1: Один ввод -- множество выходов

Учитель вводит данные **один раз** -- в любую из подключенных систем. Alashed EDU получает данные автоматически:

- Расписание -- из электронного дневника
- Список учеников -- из журнала
- Результаты оценивания -- из LMS или дневника
- Тематическое планирование -- из КТП

### Принцип 2: Двусторонний обмен

Интеграция работает в обе стороны:

- **Из дневника в AI:** расписание, списки, оценки
- **Из AI в дневник:** сгенерированные КМЖ, результаты ФО, тематические планы

### Принцип 3: Без нарушения существующих процессов

Подключение Alashed EDU **не меняет** работу существующих систем. Учителя продолжают использовать привычные инструменты, AI работает параллельно.

## Интеграция с электронным дневником

### Kundelik

Kundelik -- наиболее распространенный электронный дневник в Казахстане. Интеграция с Alashed EDU позволяет:

**Получать из Kundelik:**
- Расписание уроков учителя
- Списки учеников по классам
- Текущие оценки учеников
- Информацию о посещаемости

**Отправлять в Kundelik:**
- Тематическое планирование из КМЖ
- Результаты формативного оценивания
- Домашние задания, сгенерированные AI

### Практическая выгода

**Без интеграции:** учитель генерирует КМЖ в Alashed EDU, затем вручную переносит тему и задания в Kundelik. Двойная работа.

**С интеграцией:** учитель генерирует КМЖ, и тема урока, домашнее задание и результаты ФО автоматически синхронизируются с Kundelik. Ноль дублирования.

## Интеграция с LMS

### Что дает интеграция с LMS

- **Задания из КМЖ** автоматически создаются как задания в LMS
- **Результаты тестов** из LMS используются AI для анализа успеваемости
- **Учебные материалы** из LMS привязываются к темам уроков

### Поддерживаемые LMS

Alashed EDU поддерживает интеграцию через стандартные протоколы:

- **API-интеграция** -- для систем с открытым API
- **LTI (Learning Tools Interoperability)** -- стандартный протокол для образовательных платформ
- **Импорт/экспорт** -- для систем без API (ручная синхронизация через файлы)

## Техническая реализация

### Подключение интеграции

1. **Администратор школы** подает запрос на интеграцию через панель Alashed EDU
2. **Техническая команда** настраивает подключение (обычно 1-3 рабочих дня)
3. **Тестирование** -- проверка корректности обмена данными
4. **Запуск** -- учителя начинают использовать интеграцию

**Важно:** учителям не нужны технические навыки. Все настройки выполняются на стороне администраторов.

### Безопасность интеграции

- Все данные передаются по **зашифрованным каналам** (HTTPS/TLS)
- **Токены доступа** вместо паролей для авторизации
- **Ограниченные права** -- Alashed EDU получает только необходимые данные
- **Логирование** -- все обмены данными фиксируются для аудита

## Сценарии использования интеграции

### Сценарий 1: Утренняя подготовка учителя

Учитель открывает Alashed EDU утром. Система автоматически подтягивает расписание из дневника: сегодня 4 урока в 7А, 8Б, 9В и 10А. По каждому уроку -- тема из КТП. Учитель генерирует КМЖ для всех четырех уроков за 10 минут. Темы и задания автоматически появляются в электронном дневнике.

### Сценарий 2: После БЖБ

Учитель проводит БЖБ через LMS (тест онлайн). Результаты автоматически попадают в Alashed EDU. AI анализирует: 5 учеников не освоили ОМ-код 8.2.1.3. Система предлагает коррекционные задания для следующего урока и отмечает результаты в электронном журнале.

### Сценарий 3: Отчет для директора

Директор запрашивает отчет по качеству преподавания. Alashed EDU агрегирует данные из всех интегрированных систем: покрытие программы (из КМЖ), результаты оценивания (из LMS и дневника), активность учителей (из платформы). Один отчет вместо сбора данных из 4 систем.

## Что делать, если интеграция невозможна

Не все школы имеют техническую возможность для полной интеграции. В этом случае Alashed EDU работает **автономно**:

- Учитель вводит данные вручную (но это минимум данных)
- КМЖ экспортируются в PDF/Word и загружаются в другие системы
- Результаты оценивания вводятся вручную
- Аналитика работает на основе данных внутри платформы

**Автономный режим менее удобен, но AI-генерация работает полноценно.** Интеграция -- это дополнительное удобство, а не обязательное условие.

## Часто задаваемые вопросы

### "Интеграция стоит дополнительных денег?"

Интеграция с основными системами (Kundelik, популярные LMS) включена в стандартную подписку школы. Кастомная интеграция с редкими системами может потребовать дополнительной настройки.

### "Что если электронный дневник обновится и интеграция сломается?"

Команда Alashed EDU мониторит обновления основных партнерских систем и оперативно адаптирует интеграцию. При критических изменениях учителя будут уведомлены заранее.

### "Можно ли отключить интеграцию?"

Да, в любой момент. Данные, уже синхронизированные, останутся в обеих системах. Новая синхронизация прекратится.

## Будущее интеграций

Мы работаем над расширением интеграций:

- **Bilimland** -- полная двусторонняя интеграция
- **NIS LMS** -- для Назарбаев Интеллектуальных школ
- **Google Classroom** -- для школ, использующих экосистему Google
- **Microsoft Teams** -- для школ на платформе Microsoft

**Цель -- единое рабочее пространство учителя.** Не 5 окон в браузере, а одна платформа, которая связана со всеми системами школы. Alashed EDU стремится быть тем связующим звеном, которое объединяет разрозненные инструменты в единую экосистему.
